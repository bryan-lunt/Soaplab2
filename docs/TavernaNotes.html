Taverna Notes

Taverna used to be an important tool for accessing Soaplab services. Here we describe how you can install version 2 of Taverna Soaplab Plugin so you can access both Soaplab-2 services as well as Soaplab-1 services using Taverna.

Notes:

  • Only Taverna-1.7 is supported, Taverna-1.6 and Taverna-2.x are not supported.
  • If the Soaplab Server you are accessing has typed interface enabled you can use individual Soaplab services using their WSDL links found on your servers Spinet front page (see an example server here that has typed interface enabled). In this case you should use standard WSDL scavenger in taverna.

How to install the plugin

  • In your Taverna Workbench, select Tools->Plugin Manager->Find New Plugins->Add Plugin Site
  • Enter "http://soaplab.sourceforge.net/taverna-plugin/" as site URL and give a site name of your choice
  • You should now see Soaplab plugin listed in the new window opened, select "Taverna Soaplab Plugin 2.0.0" check box and then select the install option


    This should download and install the plugin, then you should see Soaplab plugin listed in the plugin manager screen
  • Close plugin manager
  • Restart Taverna
  • Now when you right click available processors you should see 'Add Soaplab scavenger (version 2)' option
    • If you don't see this option it means plugin not installed, check Taverna log file for errors
    • If you see the above option test it by installing a new Soaplab scavenger
  • In order to get your Soaplab2 services automatically recognized by Taverna you should add a new properties line in your ${taverna-home}/conf/mygrid.properties file, similar to the following line. 
    taverna.defaultsoaplab2 = http://localhost:8080/soaplab2/services

Known problems

  • Plugin works with both Java-5 and Java-6 but for Java-6, update-4 or later updates are required
  • When using both Java-5 and Java-6 you should either disable the Taverna 2 plugin using the Taverna plugin manager or you should manually edit the plugins/plugis.xml file (under your Taverna home directory where your Taverna Raven repository sits) and update its jaxb-api and jaxb-impl dependencies to 2.1 and 2.1.6 respectively.
