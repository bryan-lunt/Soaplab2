How to provide Soaplab2 metadata in your own way

Soaplab2 uses two kinds of metadata: one describing list of available (provided) services, and one describing individual services. By default, they are both expressed as XML files: the analysis list according to the SoaplabList.dtd, the service metadata complying with the SoaplabAnalysis.dtd.

If you want to have your own format, or storage, of any of these kinds, you can. For example, you can store the XML metadata files in the database instead of having them in the local file system. Or, you can have your own format, perhaps a non-XML one, for all or just some of your Soaplab services.

In order to accomplish it, you need to write some implementation classes. This How-To document explains what is needed and where to put it.

Actually, the main reason of this How-To document is to create a description what classes are being used and how they are connected in order to make the future Soaplab2 maintenance easier. But one never knows...
